### Gitの競合を解消する手順

①リベースを活用する
リモートの変更をローカルに取り込み、さらにローカルの変更を再適用する方法です。
これにより、リモートの変更が取り込まれた後、ローカルの変更が再適用されます。その後、変更をリモートにプッシュします。

git pull --rebase origin main
git push origin main



②変更内容を選択する
マージコンフリクトが発生した際に使用されるコマンドで、リモートの変更（theirs側の変更）を優先して、すべてのコンフリクトしたファイルをリモートの状態に上書きするためのものです。
コンフリクトが発生した場合、リモート（マージ元）の変更内容を選択します。
theirs = リモート（競合相手）
ours = 自分の変更

git checkout --theirs .
git add .
git commit
git pull
